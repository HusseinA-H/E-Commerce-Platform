Pillar Definitions

Project Overview

APEX LUXE is a next-generation luxury sportswear commerce ecosystem combining enterprise scaling, contextual AI intelligence, and multi-tenant isolation.

It establishes a unified codebase that powers completely isolated brand storefronts (SaaS), aggregates vendor products (Marketplace), and curates hyper-personalized clothing recommendations (AI Stylist).

01. Intelligent Commerce

Generative AI features powering conversational wardrobe planning and catalog indexing.

02. Enterprise Multi-Tenant SaaS

Strict data segregation and dynamic theme customization under dedicated subdomains.

03. Integrated Marketplace

Split warehouse routing and automated vendor payout logic via Stripe Connect.

Multi-Tenant SaaS

Tenant Query Isolation

Data isolation is implemented via global Prisma client query extensions. The middleware intercepts database calls, extracting the tenant domain from `AsyncLocalStorage` request headers and automatically scoping database reads, writes, and updates.

Dynamic Branding Engine

Tenant profiles define custom CSS classes, color values (primary, secondary, background), localized dictionary defaults, and domain routing. Storefront layouts adapt on the fly, rendering unique styles with zero code modifications.

APEXLUXE Multi-Tenant Architecture
Production Hardening Case Study

Engineering Challenges

Hydration Mismatch

Browser extension injection (e.g. `trancy-version`) mutated layouts before React completed rendering, causing client/server mismatch. Suppressed warnings on `html` and isolated dynamic attributes within client lifecycle hooks.

Secure Upsert Scopes

Prisma upsert constraints did not allow appending non-unique variables like `tenantId`. Built an asynchronous pre-flight client interceptor that queries the target record and blocks the execution if it belongs to another tenant.

Stripe Element State

Client-side trust models were vulnerable to payload modifications. Rebuilt payment flows using cryptographic Stripe Webhook validation, linking transitions directly to verified webhook events inside BullMQ workers.
